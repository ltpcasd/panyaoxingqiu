# 心动星球情侣小程序 - 安全机制设计方案

## 一、产品概述

### 1.1 产品定位
「心动星球」是一款专为情侣打造的私密双人空间小程序，核心特点是**仅限两人使用**的极致私密性。

### 1.2 安全目标
- **私密性**：严格限定仅情侣双方可访问
- **安全性**：防止未授权访问和数据泄露
- **隔离性**：双方数据独立存储，互不干扰
- **可控性**：用户可自主管理配对关系

### 2.3 推荐方案：邀请码 + 二维码 双模式

#### 方案A：邀请码模式（推荐）
```
┌─────────────────────────────────────────┐
│           邀请码生成规则                 │
├─────────────────────────────────────────┤
│  格式：XXXX-XXXX-XXXX（12位字母数字）    │
│  有效期：24小时                          │
│  使用次数：1次（绑定后失效）              │
│  加密方式：AES-256 + 时间戳签名          │
│  防暴力破解：5次错误输入锁定24小时        │
└─────────────────────────────────────────┘
```

**流程说明：**
1. 用户A点击"创建情侣空间"
2. 系统生成唯一邀请码，同时创建"待配对"状态的关系记录
3. 邀请码通过加密通道传输，支持复制/微信分享
4. 用户B输入邀请码后，系统验证：
   - 邀请码是否存在且未过期
   - 用户B是否已处于其他关系中
   - 用户A是否已处于其他关系中
5. 验证通过后，双方进入"确认配对"环节

#### 方案B：二维码模式
```
┌─────────────────────────────────────────┐
│           二维码生成规则                 │
├─────────────────────────────────────────┤
│  类型：动态二维码（每30秒刷新）          │
│  内容：加密的关系ID + 时间戳 + 签名      │
│  扫描方式：小程序扫码/微信扫一扫         │
│  有效期：单次有效，扫描后立即失效        │
└─────────────────────────────────────────┘
```

### 4.2 数据隔离规则

```sql
-- 数据访问控制示例（伪代码）

-- 1. 用户只能访问自己的私有数据
SELECT * FROM personal_diary 
WHERE user_id = CURRENT_USER_ID;

-- 2. 共享数据需要验证关系权限
SELECT * FROM shared_album 
WHERE relationship_id IN (
  SELECT relationship_id 
  FROM couple_relationship 
  WHERE (user_a_id = CURRENT_USER_ID OR user_b_id = CURRENT_USER_ID)
  AND status = 'active'
);

-- 3. 数据写入权限控制
INSERT INTO chat_messages (relationship_id, sender_id, content, encrypted_content)
VALUES (
  'rel_abc123', 
  CURRENT_USER_ID, 
  '[ENCRYPTED]',
  encrypt(content, SHARED_SECRET_KEY)
);
```

## 六、异常处理方案

### 6.1 边界情况处理矩阵

| 场景 | 检测机制 | 处理方式 | 用户通知 |
|-----|---------|---------|---------|
| 一方删除小程序 | 定期心跳检测 | 保留关系30天，提醒重新登录 | 推送+短信 |
| 一方更换微信 | UnionID变化检测 | 引导重新绑定，验证身份 | 双方推送 |
| 一方注销微信 | 登录态失效 | 自动解绑，数据归档 | 另一方推送 |
| 尝试第三方登录 | 设备指纹+行为分析 | 拦截并告警 | 双方推送 |
| 暴力破解邀请码 | 频率限制+验证码 | 锁定账号24小时 | 安全提醒 |
| 同时发起解绑 | 事务锁机制 | 合并处理，避免冲突 | 双方确认 |
| 网络攻击 | WAF+风控系统 | 自动防护，人工介入 | 安全公告 |

### 8.2 API安全设计

```javascript
// 中间件：关系权限验证
const coupleAuthMiddleware = async (req, res, next) => {
  const userId = req.user.id;
  const relationshipId = req.params.relationshipId || req.body.relationshipId;
  
  // 1. 验证用户是否在关系中
  const relationship = await CoupleRelationship.findOne({
    where: {
      id: relationshipId,
      status: 'active',
      [Op.or]: [
        { user_a_id: userId },
        { user_b_id: userId }
      ]
    }
  });
  
  if (!relationship) {
    return res.status(403).json({
      error: '无权访问该资源'
    });
  }
  
  // 2. 将关系信息附加到请求
  req.relationship = relationship;
  req.partnerId = relationship.user_a_id === userId 
    ? relationship.user_b_id 
    : relationship.user_a_id;
  
  next();
};

// 接口示例：发送消息
app.post('/api/messages', 
  authMiddleware,           // 验证登录
  coupleAuthMiddleware,     // 验证关系权限
  rateLimitMiddleware,      // 限流
  async (req, res) => {
    const { content } = req.body;
    
    // 使用共享密钥加密
    const encryptedContent = encrypt(content, req.relationship.shared_secret_key);
    
    const message = await Message.create({
      relationship_id: req.relationship.id,
      sender_id: req.user.id,
      receiver_id: req.partnerId,
      encrypted_content: encryptedContent,
      created_at: new Date()
    });
    
    res.json({ success: true, message });
  }
);
```

## 九、总结

### 9.1 方案亮点

1. **双模式配对**：邀请码+二维码，兼顾安全与便捷
2. **多层验证**：微信认证+设备指纹+行为分析

4. **冷静期机制**：48小时冷静期，避免冲动操作
