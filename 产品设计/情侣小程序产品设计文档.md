# 💕「心动星球」情侣小程序产品需求文档

---

## 一、产品定位与目标用户

### 1.1 产品定位
**「心动星球」** 是一款专注于**情感连接与共同成长**的情侣互动小程序，通过游戏化设计、AI情感分析和共创记忆，让情侣关系在日常互动中持续升温。

**核心理念：**
- 不只是记录，更是共同创造
- 不只是游戏，更是情感投资
- 不只是相册，更是时光胶囊

### 1.2 目标用户画像

#### 主要用户群体（18-30岁）
| 维度 | 描述 |
|------|------|
| **年龄分布** | 18-30岁（占比70%），31-35岁（占比25%） |
| **关系阶段** | 热恋期、稳定交往期、新婚期 |
| **地域分布** | 一二线城市为主，异地恋用户占比40% |
| **使用场景** | 通勤、睡前、约会间隙、纪念日准备 |

#### 典型用户画像

**用户A：小甜（24岁，女，设计师）**
- 与男友交往2年，同城但工作忙碌
- 喜欢记录生活，热衷拍照和Vlog
- 痛点：日常互动少，想增加仪式感但没时间策划
- 需求：简单易用的互动方式，自动生成的纪念内容

**用户B：阿杰（27岁，男，程序员）**
- 与女友异地恋1年，每月见面1-2次
- 不擅长表达情感，但愿意付出行动
- 痛点：不知道怎么制造惊喜，容易忘记重要日子
- 需求：智能提醒、一键生成惊喜方案、互动游戏

**用户C：晓晓&小明（新婚夫妻，28岁）**
- 刚结婚半年，正在适应婚后生活
- 希望保持恋爱感，规划共同目标
- 痛点：生活趋于平淡，缺少共同话题
- 需求：共同目标管理、新鲜感维持工具

### 2.2 🎮 星球探险 - 互动游戏中心

#### 2.2.1 默契大考验（创新版）

**传统痛点：** 市面上的默契测试题目陈旧，答完即走，缺乏持续吸引力。

**创新设计：**

**🎯 动态题库系统**
- **AI生成题目**：基于用户相册、聊天记录（授权后）生成个性化题目
  - 示例：「你们上次一起去的地方是？」「TA昨天穿的什么颜色衣服？」
- **热点题库**：结合节日、热门影视、网络梗实时更新
- **用户共创**：情侣可以互相出题，形成专属题库

**🎯 默契等级体系**
```
青铜 → 白银 → 黄金 → 铂金 → 钻石 → 星耀 → 王者
```
- 每个等级解锁新的游戏模式和特权
- 等级徽章可展示在个人主页

**🎯 游戏模式多样化**
| 模式 | 玩法 | 时长 |
|------|------|------|
| 闪电战 | 10题快问快答 | 3分钟 |
| 深度挑战 | 30题情感话题 | 15分钟 |
| 极限二选一 | 两难选择看默契 | 5分钟 |
| 你画我猜 | 绘画版默契测试 | 10分钟 |

**🎯 趣味机制**
- **默契值波动图**：记录每次测试的默契值变化
- **默契报告**：每周生成默契分析报告，指出需要加强了解的领域
- **默契任务**：根据薄弱项推荐互动任务（如"本周一起尝试一家新餐厅"）

#### 2.2.2 恋爱剧本杀（独家创新）

**核心概念：** 专为情侣设计的双人互动剧情游戏，通过角色扮演增进情感连接。

**剧本类型：**

**🎭 情感回忆本**
- 将你们的真实故事改编成剧本
- 重新演绎初遇、表白、第一次约会等场景
- 可以选择不同结局，探讨"如果当初..."

**🎭 幻想冒险本**
- 设定：穿越到古代/未来/异世界
- 任务：共同解谜、克服困难
- 特色：剧情分支由双方选择决定，影响最终结局

**🎭 悬疑推理本**
- 扮演侦探搭档破解谜案
- 需要双方协作收集线索
- 考验沟通能力和信任度

**游戏流程：**
```
选择剧本 → 分配角色 → 阅读背景 → 互动推进 → 共同决策 → 解锁结局 → 生成剧照明信片
```

**创新点：**
- **AI主持人**：智能推进剧情，根据玩家反应调整难度
- **情感洞察**：游戏结束后分析双方的决策模式，给出关系建议
- **剧本创作器**：用户可以创作自己的剧本并分享

#### 2.2.3 双人协作闯关

**游戏设计：**

**🏰 星球探险地图**
- 虚拟地图分为多个主题区域（甜蜜森林、勇气火山、智慧海洋等）
- 每个区域包含10-15个关卡
- 关卡类型：解谜、协作、知识问答、动作挑战

**🎮 关卡示例：**

| 关卡名 | 类型 | 玩法说明 |
|--------|------|----------|
| 同步心跳 | 协作 | 双方需在3秒内同时点击屏幕，考验默契 |
| 记忆拼图 | 解谜 | 各自掌握一半信息，通过语音沟通完成拼图 |
| 知识擂台 | 问答 | 关于对方的问题，答对为团队积累分数 |
| 勇气试炼 | 挑战 | 完成真心话大冒险任务才能通关 |

**奖励系统：**
- 通关获得"星尘"货币
- 星尘可兑换虚拟装饰、特权功能、实体礼品折扣
- 连续登录解锁限定关卡

#### 2.2.4 心情盲盒

**概念：** 每日情绪交换的趣味方式

**玩法：**
1. 每天双方各自选择一个表情代表当天心情
2. 放入"盲盒"，对方在特定时间才能开启
3. 开启后可以看到对方心情，并选择回应方式
4. 连续7天交换解锁"心情周报"

**创新点：**
- **心情预测**：AI根据历史数据预测对方今日心情
- **智能建议**：根据心情推荐互动方式（如"TA今天累了，发个抱抱吧"）
- **心情地图**：月度心情可视化，发现情绪规律

### 2.3 📅 时光轴 - 纪念日管理

#### 2.3.1 智能纪念日系统

**传统痛点：** 手动添加繁琐，容易遗漏重要日子。

**创新功能：**

**🤖 AI自动识别**
- 分析聊天记录中的日期信息，智能推荐纪念日
- 识别相册照片时间，提示"3年前的今天你们..."
- 自动计算交往天数、结婚周年等

**📊 纪念日矩阵**
```
一级纪念日：交往日、结婚日、双方生日（提前7天提醒）
二级纪念日：第一次约会、第一次旅行、见家长日（提前3天提醒）
三级纪念日：自定义特殊日子（提前1天提醒）
隐藏纪念日：AI发现的有趣日子（如"连续聊天100天"）
```

**🎨 视觉设计**
- 时间轴形式展示所有纪念日
- 每个纪念日有专属图标和配色
- 支持上传照片、视频、语音作为纪念内容

#### 2.3.2 时光胶囊（核心创新）

**概念：** 写给未来的信，在特定日期自动开启

**功能设计：**

**💊 胶囊类型**
| 类型 | 说明 | 示例 |
|------|------|------|
| 时间胶囊 | 设定未来日期开启 | "一年后打开，看看我们的愿望实现了吗" |
| 条件胶囊 | 达成条件后开启 | "当我们一起去过10个城市后开启" |
| 随机胶囊 | 随机时间开启 | 给未来的对方一个惊喜 |

**胶囊内容形式：**
- 文字信件
- 语音留言
- 视频录像
- 照片集
- 混合多媒体

**特色功能：**
- **双人胶囊**：双方各自写内容，到时间同时开启
- **胶囊树**：可视化展示所有胶囊，像挂果实的树
- **胶囊提醒**：开启前3天通知双方准备

#### 2.3.3 未来约定

**功能：** 共同规划未来的工具

**模块设计：**

**🗺️ 愿望地图**
- 世界地图标记想一起去的地方
- 每个地点可添加愿望清单
- 完成后点亮，形成专属旅行足迹

**📋 共同清单**
- 100件小事模板（可自定义）
- 进度追踪和完成打卡
- 完成奖励和纪念

**⏰ 倒计时器**
- 重要事件倒计时（如"距离见面还有3天"）
- 支持自定义背景和音乐
- 桌面小组件支持

#### 2.3.4 关系健康度（AI分析）

**概念：** 通过数据分析关系状态，提供建议

**评估维度：**
```
沟通频率：聊天、语音、视频通话统计
互动质量：游戏参与度、任务完成率
情感表达：情话发送、礼物记录、惊喜次数
共同成长：目标达成、新体验数量
```

**输出内容：**
- 每周健康度评分（1-100分）
- 趋势分析图表
- 个性化改进建议
- 关系里程碑提醒

**隐私保护：**
- 所有分析仅在本地进行
- 可选择关闭此功能
- 数据不上传云端

### 2.4 📸 记忆星云 - 相册系统

#### 2.4.1 AI相册助手

**核心功能：**

**🤖 智能整理**
- 自动识别合照、风景、美食等分类
- 人脸识别标记
- 重复照片清理建议
- 照片质量评分（模糊、过曝提醒）

**✨ AI增强**
- 老照片修复
- 黑白照片上色
- 合照补全（如果某人闭眼，AI尝试修复）
- 风格迁移（照片变油画、漫画等）

**📝 智能配文**
- AI根据照片内容生成文案
- 提供多种风格选择（文艺、幽默、甜蜜、简洁）
- 支持自定义模板

#### 2.4.2 故事时间线

**创新设计：**

**📖 自动故事生成**
- 选择时间段，AI自动生成故事线
- 包含照片、地点、时间、配文
- 可编辑调整，添加遗漏内容

**🎬 动态影集**
- 选择模板，一键生成视频
- 支持添加音乐、转场效果
- 多种主题：恋爱周年、旅行回忆、年度总结

**📅 日历视图**
- 以日历形式展示照片
- 点击日期查看当天所有照片
- 空白日期提醒"今天还没有记录哦"

#### 2.4.3 共创影集

**概念：** 双方共同编辑的专属相册

**协作功能：**
- 实时协作编辑（类似腾讯文档）
- 评论和标注功能
- 版本历史记录
- 编辑权限管理

**影集类型：**
| 类型 | 用途 |
|------|------|
| 旅行手账 | 记录每次旅行 |
| 美食地图 | 打卡过的餐厅 |
| 穿搭进化史 | 记录风格变化 |
| 宠物成长记 | 记录毛孩子日常 |

#### 2.4.4 私密空间

**功能设计：**

**🔒 隐私保护**
- 独立密码/指纹解锁
- 内容不上传云端，仅本地存储
- 应用切换时自动模糊

**💝 私密内容**
- 私密照片/视频
- 悄悄话记录
- 只有两人知道的秘密
- 加密日记

### 2.5 🎯 成长任务 - 共同目标

#### 2.5.1 情侣成就系统

**设计思路：** 游戏化激励，让日常互动变得有趣

**成就分类：**
```
💬 沟通类：连续聊天7天、深夜谈心、化解矛盾
🎮 互动类：游戏通关、完成任务、创造回忆
🎁 表达类：发送情话、准备惊喜、手写情书
🌟 成长类：共同学习、养成习惯、达成目标
```

**等级体系：**
- 每个成就有铜、银、金、钻石四个等级
- 累计成就点提升"情侣等级"
- 高等级解锁专属称号和特权

**展示方式：**
- 成就墙展示已获得成就
- 可分享至朋友圈（脱敏处理）
- 成就徽章收集册

#### 2.5.2 100件小事

**功能设计：**

**📋 清单模板**
- 提供10+主题清单模板
- 每类100件事，可自定义增删
- 支持创建自己的专属清单

**模板示例：**
| 主题 | 示例内容 |
|------|----------|
| 恋爱必做 | 一起看日出、为对方做早餐、说早安晚安100天 |
| 城市探索 | 去城市最高点、逛夜市、坐摩天轮 |
| 技能学习 | 一起学做一道菜、学一支舞、学一门外语 |
| 冒险挑战 | 蹦极、潜水、徒步、露营 |

**打卡机制：**
- 完成打卡，上传照片/视频证明
- 双方确认后生效
- 可添加心得和评分

#### 2.5.3 习惯养成

**功能：** 共同培养好习惯

**习惯类型：**
- 个人习惯：早睡早起、运动、阅读
- 共同习惯：每天说早安、每周约会、每月旅行

**游戏化设计：**
- 连续打卡获得"连击"奖励
- 中断后可通过"补签卡"恢复（需完成任务获得）
- 习惯排行榜（可选参与）
- 习惯达成奖励

#### 2.5.4 梦想清单

**功能：** 管理长期共同目标

**目标管理：**
- 设定目标（买房、创业、环球旅行等）
- 分解为可执行步骤
- 设置里程碑和截止日期
- 进度追踪和可视化

**协作功能：**
- 双方各自的任务分配
- 进度同步和提醒
- 达成庆祝仪式

### 2.6 💌 心动信箱 - 情感表达

#### 2.6.1 匿名真心话

**玩法：**
- 每周可发送1-3个匿名问题给对方
- 对方回答后，可选择是否揭晓身份
- 问题类型：情感、生活、未来、趣味

**问题库：**
- 系统提供精选问题
- 用户可自定义问题
- 热门问题排行榜

**特色：**
- 回答后可看到对方是否猜中是你问的
- 积累"真心话积分"解锁高级问题

#### 2.6.2 语音情书

**功能：**
- 录制语音消息，设定送达时间
- 可选择背景音乐和特效
- 支持定时发送（如"每天早上8点发送早安语音"）

**创新点：**
- **语音变声**：可爱、磁性、搞笑等效果
- **语音合成**：文字转语音，选择不同声音风格
- **语音盲盒**：随机发送给对方的语音惊喜

#### 2.6.3 惊喜任务

**概念：** 系统推荐的惊喜方案

**任务类型：**
| 难度 | 任务示例 | 预计时间 |
|------|----------|----------|
| 简单 | 发一条仅TA可见的朋友圈 | 5分钟 |
| 中等 | 手写一封信拍照发送 | 30分钟 |
| 困难 | 准备一次浪漫晚餐 | 3小时 |
| 挑战 | 策划一次短途旅行 | 1周 |

**个性化推荐：**
- 基于对方喜好推荐
- 基于纪念日推荐
- 基于关系健康度推荐

#### 2.6.4 每日情话

**功能：**
- 每天推送精选情话
- 可一键发送给对方
- 支持自定义收藏

**内容来源：**
- 经典文学、电影台词
- 用户投稿精选
- AI生成个性化情话

**互动：**
- 收到情话可回复表情或语音
- 情话收藏夹
- 情话排行榜

## 三、用户旅程地图

### 3.1 新用户旅程

```
【发现阶段】
用户触点：朋友圈分享、应用商店、情侣博主推荐
用户行为：看到有趣的分享内容，产生兴趣
用户想法："这个看起来挺有意思的，想和TA试试"
痛点：担心操作复杂、隐私问题
机会点：强调简单易用、隐私安全

【注册阶段】
用户触点：小程序入口
用户行为：授权登录，邀请另一半
用户想法："希望TA也能接受这个"
痛点：邀请流程复杂、对方不响应
机会点：简化邀请流程，提供邀请话术模板

【首次体验】
用户触点：新手引导
用户行为：完成默契测试，获得第一个成就
用户想法："原来还可以这样玩，挺有趣的"
痛点：功能太多不知道从何开始
机会点：智能推荐最适合的功能

【激活阶段】
```

## 五、差异化竞争优势

### 5.1 与竞品对比分析

| 维度 | 传统情侣App | 心动星球 |
|------|-------------|----------|
| **核心定位** | 工具型（记录、提醒） | 情感型（连接、成长） |
| **互动方式** | 单向记录为主 | 双向游戏化互动 |
| **内容生产** | UGC为主，质量参差 | AIGC+PGC+UGC结合 |
| **情感深度** | 浅层记录 | 深度情感分析 |
| **新鲜感维持** | 功能固定，易厌倦 | 持续更新，动态内容 |
| **社交属性** | 封闭私密 | 可控分享，社区潜力 |

### 5.2 核心竞争优势

#### 1. 🎮 游戏化情感连接
**竞品现状：** 大多数情侣App是工具属性，用完即走
**我们的创新：** 
- 将情感维护游戏化，让"经营感情"变得有趣
- 通过游戏机制自然引导深度交流
- 成就系统和等级体系提供持续动力

#### 2. 🤖 AI驱动的个性化体验
**竞品现状：** 千篇一律的功能和内容
**我们的创新：**
- AI生成个性化题目、情话、剧本
- 智能分析关系健康度，提供针对性建议
- 基于用户数据的智能推荐

#### 3. 📖 独家恋爱剧本杀
**竞品现状：** 无类似功能
**我们的创新：**
- 市场上首个专为情侣设计的剧本杀
- 结合真实故事的定制化剧本
- 角色扮演增进情感连接

#### 4. 💊 时光胶囊情感投资
**竞品现状：** 简单的纪念日提醒
**我们的创新：**
- 写给未来的信，创造期待感
- 条件触发机制，增加趣味性
- 情感投资的具象化体现

#### 5. 🌱 共同成长导向
**竞品现状：** 聚焦当下，忽视长期
**我们的创新：**
- 100件小事、习惯养成、梦想清单
- 帮助情侣建立共同目标和成长路径
- 从"记录感情"到"经营感情"再到"共同成长"

## 六、UI设计指导建议

### 6.1 视觉风格定位

**整体风格：** 梦幻宇宙 × 甜蜜恋爱

**色彩系统：**
```
主色调：
- 星空紫：#6B5B95（神秘、浪漫）
- 心动粉：#FF6B9D（甜蜜、温暖）

辅助色：
- 月光白：#F8F9FA（纯净、简洁）
- 星云蓝：#4ECDC4（清新、希望）
- 暖阳黄：#FFE66D（活力、快乐）

强调色：
- 爱心红：#FF4757（重要提醒、特殊时刻）
```

**字体规范：**
- 标题：圆润无衬线字体（如PingFang SC Medium）
- 正文：清晰易读无衬线字体（如PingFang SC Regular）
- 特殊：手写体用于情话、信件等情感内容

**图形风格：**
- 图标：线性+填充结合，圆角设计
- 插画：扁平化星球、星座、宇宙元素
- 动效：流畅的过渡动画，星星闪烁、流星划过等

### 6.2 关键页面设计要点

#### 首页（星球导航）
- 背景：动态星空，可交互的星球
- 布局：5大功能模块以星球形式分布
- 互动：点击星球进入对应功能，有引力动画效果
- 状态：显示今日任务进度、连续登录天数

#### 默契大考验页面
- 氛围：竞赛感但保持轻松
- 题目展示：大字体，清晰易读
- 答题交互：双方同时显示选项，倒计时动画
- 结果展示：默契值动画增长，解锁成就提示

#### 时光轴页面
- 布局：垂直时间线，左右交替展示
- 节点：重要日期放大显示，有光晕效果
- 交互：点击展开详细内容，支持滑动浏览
- 空状态：温馨的引导，鼓励添加第一个纪念日

#### 记忆星云页面
- 布局：瀑布流/网格视图切换
- 照片展示：圆角卡片，悬停显示时间地点
- AI功能入口：悬浮按钮，一键优化
- 故事线：时间轴+地图双视图

### 6.3 交互设计原则

1. **即时反馈**：每个操作都有视觉/听觉反馈
2. **情感化设计**：加载动画、空状态都要传递温暖
3. **双人视角**：同时考虑两个用户的使用场景
4. **游戏化引导**：新手引导以"星球探险"故事形式展开
5. **隐私保护**：敏感操作需要二次确认，隐私内容有明显标识

## 七、风险评估与应对

### 7.1 主要风险

| 风险类型 | 具体风险 | 应对措施 |
|----------|----------|----------|
| 产品风险 | 情侣分手导致用户流失 | 提供"冷静期"功能，支持关系修复内容 |
| 产品风险 | 新鲜感消退，用户流失 | 持续更新内容，建立内容生产机制 |
| 技术风险 | AI生成内容质量不稳定 | 人工审核+AIGC结合，建立质量评估体系 |
| 运营风险 | 用户隐私泄露 | 端到端加密，本地化存储，严格权限管理 |
| 商业风险 | 付费意愿低 | 免费功能足够好用，付费提供增值体验 |
| 竞争风险 | 大厂入场 | 深耕细分场景，建立用户心智和品牌认知 |

### 7.2 成功关键因素

1. **内容质量**：剧本杀、题目等内容必须高质量、常更新
2. **技术体验**：AI功能要真正有用，不能是噱头
3. **用户隐私**：建立绝对信任，隐私保护是底线
4. **社区氛围**：如果开放社区，必须维护正向氛围
5. **运营能力**：持续的内容生产和用户活动运营
