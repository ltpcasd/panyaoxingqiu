# 联调测试执行指南

## 🚀 第一步：环境准备

### 1. 检查前置依赖

```powershell
# 检查Node.js版本（需要v18+）
node -v

# 检查MySQL服务状态
Get-Service -Name MySQL*

# 检查Redis服务状态（如果安装为Windows服务）
Get-Service -Name Redis*
```

### 2. 创建后端环境配置

在 `后端\panyaoxingqiu-server\` 目录下创建 `.env` 文件：

```env
# 服务配置
NODE_ENV=development
PORT=3000
API_PREFIX=api/v1

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=你的MySQL密码
DB_DATABASE=panyaoxingqiu

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

# JWT配置
JWT_SECRET=panyaoxingqiu-jwt-secret-key-for-dev
JWT_EXPIRES_IN=7d

# 微信小程序配置
WECHAT_APPID=你的小程序AppID
WECHAT_SECRET=你的小程序Secret

# 微信小程序配置（测试环境可使用测试号）
# 开发阶段可以跳过微信登录验证

# 文件上传配置
UPLOAD_DIR=./uploads
```

### 3. 初始化数据库

```powershell
# 进入后端目录
cd "d:\ai项目\ai-test-demo\后端\panyaoxingqiu-server"

# 创建数据库（如果未创建）
# 在MySQL中执行：
# CREATE DATABASE IF NOT EXISTS panyaoxingqiu DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 运行数据库迁移（如果有）
# npm run migration:run

# 或者使用TypeORM自动同步（开发环境）
# synchronize: true 已在配置中开启
```

### 4. 启动后端服务

```powershell
cd "d:\ai项目\ai-test-demo\后端\panyaoxingqiu-server"

# 安装依赖
npm install

# 启动开发服务器
npm run start:dev
```

**启动成功标志**：
- 控制台输出：🚀 潘瑶星球后端服务启动成功: http://localhost:3000/api/v1
- 控制台输出：📚 API文档地址: http://localhost:3000/api/v1/docs
- 访问 http://localhost:3000/api/v1/docs 可以看到Swagger文档

### 5. 启动前端服务

```powershell
# 新开一个终端窗口
cd "d:\ai项目\ai-test-demo\前端\panyaoxingqiu-miniapp"

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

**启动成功标志**：
- H5版本：浏览器自动打开，或访问 http://localhost:5173
- 微信小程序：使用微信开发者工具导入项目

## 🧪 第二步：接口测试（使用Swagger）

### 测试方法

1. 访问 Swagger 文档：http://localhost:3000/api/v1/docs
2. 先获取测试用的Token
3. 点击右上角 "Authorize" 按钮，输入Token
4. 测试各个接口

### 2.1 测试认证模块

#### 模拟微信登录（开发环境）

由于微信登录需要真实的code，开发环境可以使用模拟登录：

**方式一：使用测试Token生成接口（如果后端有提供）**

```http
POST http://localhost:3000/api/v1/auth/login
Content-Type: application/json

{
  "code": "test_code_001",
  "userInfo": {
    "nickName": "测试用户A",
    "avatarUrl": "https://via.placeholder.com/100",
    "gender": 1
  }
}
```

**方式二：直接在数据库插入测试用户**

```sql
-- 在MySQL中执行
USE panyaoxingqiu;

-- 插入测试用户
INSERT INTO users (id, openid, nickname, avatar, gender, created_at, updated_at) 
VALUES 
(UUID(), 'test_openid_001', '测试用户A', 'https://via.placeholder.com/100', 1, NOW(), NOW()),
(UUID(), 'test_openid_002', '测试用户B', 'https://via.placeholder.com/100', 2, NOW(), NOW());
```

**方式三：使用JWT生成测试Token**

使用在线工具或代码生成JWT Token：
- Payload: { "userId": "用户ID", "openid": "test_openid_001" }
- Secret: panyaoxingqiu-jwt-secret-key-for-dev
- ExpiresIn: 7d

## ⚠️ 常见问题排查

### 后端服务启动失败

1. **数据库连接失败**
   - 检查MySQL服务是否启动
   - 检查数据库用户名密码
   - 检查数据库是否已创建

2. **Redis连接失败**
   - 检查Redis服务是否启动
   - 检查Redis端口是否正确

3. **端口被占用**
   ```powershell
   # 查看端口占用
   netstat -ano | findstr :3000
   
   # 结束占用进程
   taskkill /PID {进程ID} /F
   ```

### 前端请求失败

1. **CORS跨域问题**
   - 检查后端CORS配置
   - 确认前端API地址正确

2. **Token失效**
   - 检查Token是否过期
   - 重新登录获取新Token

3. **接口404**
   - 检查接口路径是否正确
   - 确认后端服务已启动
