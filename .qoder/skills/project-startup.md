# 项目启动引导

## 触发条件

当用户说"我要做一个程序"、"我想开发一个应用"、"帮我做一个项目"等类似表述时触发。

---

## 引导流程

### 第一步：询问程序类型

**标准提问模板**：
> 请告诉我您想做什么类型的程序？
> 
> **平台类型**：
> - Web应用（浏览器访问，PC+移动端兼容）
> - PC网页（桌面端网站、管理系统、官网）
> - H5页面（移动端网页、营销活动页）
> - 微信小程序（微信内运行）
> - 移动App（iOS/Android原生或跨平台）
> - 桌面应用（Windows/Mac客户端）
> - 后端服务（API服务、微服务）
> - 其他类型
> 
> 请选择或描述您的程序类型：

---

### 第二步：市场定位和样式建议

根据用户选择的类型，提供市场和样式建议：

#### 2.1 市场类型建议

**根据程序类型推荐市场方向**：

| 程序类型 | 常见市场方向 |
|----------|-------------|
| Web应用 | 企业管理、电商平台、SaaS服务、社区论坛、在线教育 |
| PC网页 | 企业官网、管理后台、数据大屏、文档中心 |
| H5页面 | 营销活动、产品介绍、问卷表单、小游戏 |
| 小程序 | 工具类、电商类、生活服务、社交娱乐 |
| 移动App | 社交、工具、电商、内容社区 |
| 桌面应用 | 办公工具、设计软件、开发工具、游戏 |

**提问模板**：
> 您的项目属于哪个市场方向？
> 
> 推荐：【根据程序类型推荐3-5个方向】
> 
> 或者您可以描述目标用户和使用场景：

#### 2.2 样式风格建议

**常见设计风格选项**：

| 风格类型 | 特点 | 适用场景 |
|----------|------|----------|
| 现代简约 | 干净、留白、清爽 | 企业站、工具类、SaaS |
| 商务专业 | 深色、稳重、高端 | 金融、企业服务、管理后台 |
| 活泼年轻 | 色彩丰富、圆润 | 社交、娱乐、教育 |
| 科技未来 | 深色、渐变、光效 | 科技产品、数据展示 |
| 温暖情感 | 柔和、暖色调 | 生活服务、社交、情感类 |
| 极简扁平 | 纯色、图标化 | 工具类、效率应用 |

**提问模板**：
> 您希望的设计风格是？
> 
> 推荐：【根据市场类型推荐2-3个风格】
> 
> 或者您可以描述期望的视觉感受：

---

### 第三步：动态丰富Agent能力 ⭐

**这是关键步骤**：根据用户描述的产品需求，自动为所有Agent补充相关行业经验。

#### 3.1 识别产品特征

根据用户输入，识别：
- 产品类型（电商/社交/企业服务/内容平台/工具/金融等）
- 目标用户（C端用户/B端企业/特定人群）
- 核心功能（交易/内容/社交/管理/工具等）
- 特殊要求（高并发/实时性/安全性/合规性等）

#### 3.2 动态能力分配表

根据产品类型，为各Agent分配专业能力：

| 产品类型 | @产品 能力 | @UI 能力 | @前端 能力 | @后端 能力 | @测试 能力 | @运维 能力 |
|----------|-----------|----------|-----------|-----------|-----------|-----------|
| **电商平台** | 商品体系、购物车、订单流程、营销活动 | 信任感设计、商品展示优化 | 性能优化、SEO、支付对接 | 高并发、库存、订单、支付安全 | 订单流程、支付安全、并发测试 | 弹性伸缩、CDN、秒杀支持 |
| **社交应用** | 用户关系、内容发布、互动机制、增长策略 | 年轻活力、互动元素、社交氛围 | 实时通信、WebSocket、消息列表 | IM系统、推送、在线状态 | 消息功能、IM稳定性、推送测试 | 实时性保障、消息队列 |
| **企业系统** | 权限管理、工作流、数据报表、多租户 | 简约高效、数据可视化 | 组件复用、表单验证、权限控制 | 权限系统、审计日志、多租户 | 权限控制、数据安全、隔离测试 | 私有部署、数据备份 |
| **内容平台** | 内容创作、推荐算法、创作者体系、社区运营 | 清爽聚焦、阅读体验 | 长列表优化、富文本、图片懒加载 | 缓存策略、CDN、搜索推荐 | 内容展示、搜索准确性、推荐质量 | CDN加速、静态资源 |
| **工具应用** | 核心功能、效率优化、数据管理 | 极简直观、操作便捷 | 离线能力、PWA、本地存储 | 轻量API、数据处理 | 核心功能、兼容性、稳定性 | 低成本、易维护 |
| **金融系统** | 账户体系、交易流程、风控设计、合规要求 | 专业稳重、数据安全 | 安全加密、数据校验 | 高可靠、事务、风控、审计 | 交易安全、风控逻辑、合规测试 | 多机房、灾备、安全加固 |
| **二手车平台** | 车源管理、估价体系、交易流程、信任建设 | 信任感设计、车辆展示 | 图片处理、筛选组件、地图集成 | 车源搜索、估价算法、交易管理 | 车源数据、交易流程、估价准确性 | 图片存储、搜索优化 |

#### 3.3 能力丰富方式

**自动执行**：
1. 分析用户描述的产品需求
2. 识别产品类型和特征
3. 根据上表为各Agent动态补充能力
4. 在后续工作中自动应用这些能力

**无需询问用户**：能力丰富是自动的，用户无感知

---

### 第四步：技术选型建议

**技术选型原则（必须遵循）**：

1. **兼容性高**：支持主流浏览器/设备/系统
2. **稳定性高**：成熟框架、社区活跃、长期维护
3. **运行效率高**：性能优秀、资源占用低

**技术栈推荐表**：

| 程序类型 | 推荐前端技术 | 推荐后端技术 | 理由 |
|----------|-------------|-------------|------|
| Web应用 | Vue3 + TypeScript / React + TypeScript | Node.js(NestJS) / Go / Java | 跨平台、生态完善、性能优秀 |
| PC网页 | Vue3 + TypeScript | Node.js / Go / Java | 兼容性好、开发效率高 |
| H5页面 | Vue3 / React / 原生JS | Node.js / Go | 轻量、加载快、兼容移动端 |
| 微信小程序 | uni-app(Vue3) / 原生小程序 | Node.js(NestJS) / Go | 跨平台、微信生态兼容 |
| 移动App | Flutter / React Native / uni-app | Node.js / Go / Java | 跨平台、一套代码多端运行 |
| 桌面应用 | Electron / Tauri | Node.js / Go / Rust | 跨平台、开发效率高 |
| 后端服务 | - | Go / Node.js / Java | 高性能、生态完善 |

**自动记录**：根据用户选择，自动确定技术栈，无需额外询问。

---

### 第五步：确认后启动工作流

**只在以下重要节点询问用户**：

| 节点 | 询问内容 |
|------|----------|
| 需求确认 | 确认程序类型、市场方向、设计风格 |
| 产品方案 | 确认PRD文档和功能列表 |
| UI设计 | 确认页面设计稿 |
| 技术方案 | 仅当有多种可行方案时询问选择 |
| 上线部署 | 确认部署方案 |

**确认模板**：
> 好的，我已了解您的需求：
> 
> **项目信息**：
> - 类型：【程序类型】
> - 方向：【市场方向】
> - 风格：【设计风格】
> - 核心功能：【功能列表】
> 
> **技术方案**：
> - 前端：【推荐技术栈】（兼容性高、稳定性高、效率高）
> - 后端：【推荐技术栈】
> 
> **能力已丰富**：
> - @产品：已补充【行业】产品经验
> - @UI：已补充【行业】设计经验
> - @前端：已补充【行业】开发经验
> - @后端：已补充【行业】开发经验
> - @测试：已补充【行业】测试经验
> - @运维：已补充【行业】运维经验
> 
> 接下来流程：
> 1. @产品 输出产品设计 → 您确认
> 2. @UI 输出页面设计 → 您确认
> 3. 研发开发 → 测试验收 → 部署上线
> 
> 确认无误后开始。是否继续？

## 注意事项

1. **程序类型不约束**：支持任何类型的程序开发
2. **技术选型三原则**：兼容性高、稳定性高、运行效率高
3. **主动给出建议**：市场方向和样式选择要主动推荐
4. **自动丰富能力**：根据产品类型自动为各Agent补充能力
5. **重要节点询问**：只在关键确认点询问，不过度打扰
