# 潘瑶星球项目部署文档

## 概述

本文档指导如何将潘瑶星球项目从Railway/Vercel迁移到阿里云，确保国内用户可以直接访问。

**部署架构：**
- **后端**：阿里云函数计算FC (NestJS + SQLite)
- **前端**：阿里云OSS静态网站托管 (uni-app H5)
- **数据库**：NAS文件存储（SQLite）或阿里云RDS MySQL

---

## 一、准备工作

### 1.1 注册阿里云账号

1. 访问 [阿里云官网](https://www.aliyun.com/)
2. 注册账号并完成实名认证
3. 开通以下服务：
   - 函数计算FC
   - 对象存储OSS
   - 文件存储NAS（如使用SQLite）
   - 云解析DNS（如使用自定义域名）

## 二、后端部署（函数计算FC）

### 2.1 项目准备

#### 修改NestJS应用以适配FC

1. **修改入口文件** (`main.ts`)

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  
  // 启用CORS
  app.enableCors({
    origin: '*', // 生产环境建议指定具体域名
    methods: 'GET,HEAD,PUT,PATCH,POST,DELETE,OPTIONS',
    credentials: true,
  });
  
  // 设置API前缀
  app.setGlobalPrefix('api/v1');
  
  const port = process.env.PORT || 9000;
  await app.listen(port);
  
  console.log(`Application is running on: ${await app.getUrl()}`);
}

// 阿里云FC需要导出handler
const server = bootstrap();

// 健康检查接口
export const handler = async (event, context) => {
  return {
    statusCode: 200,
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      status: 'ok',
      timestamp: new Date().toISOString(),
    }),
  };
};
```

2. **修改数据库配置** (如果使用SQLite)

```typescript
// app.module.ts
import { TypeOrmModule } from '@nestjs/typeorm';

@Module({
  imports: [
    TypeOrmModule.forRoot({
      type: 'sqlite',
      database: process.env.DATABASE_PATH || './data/database.sqlite',
      entities: [__dirname + '/**/*.entity{.ts,.js}'],
      synchronize: true, // 生产环境建议关闭，使用迁移
    }),
    // ...其他模块
  ],
})
export class AppModule {}
```

3. **创建构建脚本** (`package.json`)

```json
{
  "scripts": {
    "build": "nest build",
    "build:prod": "nest build --webpack --webpackPath webpack.config.js",
    "start:prod": "node dist/main"
  }
}
```

### 2.2 数据库方案选择

#### 方案A：使用NAS存储SQLite（推荐，成本最低）

**优点：**
- 免费额度内使用
- 无需修改现有代码
- 数据持久化存储

**缺点：**
- 并发性能有限
- 不适合高并发场景

**配置步骤：**

1. 创建NAS文件系统
   - 进入 [NAS控制台](https://nas.console.aliyun.com/)
   - 创建文件系统（选择"通用型NAS"）
   - 添加挂载点
   - 记录挂载地址

2. 修改 `aliyun-fc.yml` 中的NAS配置

```yaml
nas:
  component: nas
  props:
    region: cn-hangzhou
    vpcId: vpc-xxx  # 你的VPC ID
    vSwitchId: vsw-xxx  # 你的交换机ID
    mountPoints:
      - mountDir: /mnt/data
        serverAddr: xxx.cn-hangzhou.nas.aliyuncs.com:/  # 你的NAS挂载地址
```

#### 方案B：使用阿里云RDS MySQL

**优点：**
- 高性能
- 支持高并发
- 自动备份

**缺点：**
- 需要付费
- 需要修改代码

**配置步骤：**

1. 创建RDS实例
   - 进入 [RDS控制台](https://rds.console.aliyun.com/)
   - 创建MySQL实例
   - 创建数据库和用户

2. 修改数据库配置

```typescript
TypeOrmModule.forRoot({
  type: 'mysql',
  host: process.env.DB_HOST,
  port: 3306,
  username: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  entities: [__dirname + '/**/*.entity{.ts,.js}'],
  synchronize: false, // 生产环境关闭
}),
```

3. 修改 `aliyun-fc.yml` 添加环境变量

```yaml
environmentVariables:
  DB_HOST: rm-xxx.mysql.rds.aliyuncs.com
  DB_USER: your_username
  DB_PASSWORD: your_password
  DB_NAME: panyaoxingqiu
```

## 三、前端部署（OSS静态托管）

### 3.1 修改前端配置

1. **修改API基础URL** (`manifest.json` 或配置文件)

```json
{
  "name": "潘瑶星球",
  "appid": "__UNI__XXXXXXX",
  "description": "潘瑶星球 - 互动故事平台",
  "versionName": "1.0.0",
  "versionCode": "100",
  "networkTimeout": {
    "request": 10000
  }
}
```

2. **修改请求基地址** (在main.js或api配置文件中)

```javascript
// 根据环境判断API地址
const API_BASE_URL = process.env.NODE_ENV === 'production' 
  ? 'https://your-fc-url.cn-hangzhou.fc.aliyuncs.com/api/v1'
  : 'http://localhost:3000/api/v1';

uni.request({
  url: `${API_BASE_URL}/your-endpoint`,
  // ...
});
```

3. **配置路由模式** (`pages.json`)

```json
{
  "pages": [
    // ...页面配置
  ],
  "globalStyle": {
    "navigationStyle": "custom"
  },
  "condition": {}
}
```

### 3.2 构建项目

```powershell
# 进入前端目录
cd 前端\panyaoxingqiu-miniapp

# 安装依赖
npm install

# 构建H5版本
npm run build:h5

# 或使用HBuilderX构建
# 1. 打开HBuilderX
# 2. 选择 发行 → 网站-H5手机版
# 3. 等待构建完成
```

构建完成后，文件位于 `dist/build/h5` 目录。

### 3.3 部署到OSS

#### 方式一：使用PowerShell脚本（推荐）

```powershell
# 运行部署脚本
.\deploy-oss.ps1
```

#### 方式二：手动部署

1. 进入 [OSS控制台](https://oss.console.aliyun.com/)
2. 创建Bucket
   - Bucket名称：全局唯一（如 `panyaoxingqiu-web-2024`）
   - 区域：与FC相同（cn-hangzhou）
   - 存储类型：标准存储
   - 读写权限：公共读

3. 配置静态网站托管
   - 默认首页：`index.html`
   - 默认404页：`index.html`（支持SPA路由）

4. 配置CORS
   - 来源：`*`（或你的域名）
   - 允许Methods：全部
   - 允许Headers：`*`

5. 上传文件
   - 进入Bucket → 文件管理
   - 上传 `dist/build/h5` 目录下的所有文件

### 3.4 配置CDN加速（可选但推荐）

1. 进入 [CDN控制台](https://cdn.console.aliyun.com/)
2. 添加域名
   - 加速域名：`www.panyaoxingqiu.com`
   - 源站类型：OSS域名
   - 源站地址：`panyaoxingqiu-web.oss-cn-hangzhou.aliyuncs.com`

3. 配置HTTPS
   - 申请免费SSL证书（阿里云提供免费证书）
   - 开启HTTPS加速

4. 配置缓存规则
   - HTML文件：不缓存
   - JS/CSS/图片：缓存1天

5. 修改DNS解析
   - 添加CNAME记录指向CDN提供的域名

## 八、常见问题

### Q1: 函数计算冷启动慢怎么办？

**解决方案：**
1. 使用预留实例（付费）
2. 优化代码启动时间
3. 使用单实例多并发
4. 配置定时触发器保持实例活跃

### Q2: SQLite在FC中数据丢失？

**原因：** FC实例是临时的，本地文件不会持久化

**解决方案：**
1. 使用NAS挂载存储
2. 改用RDS数据库
3. 定期备份到OSS

### Q3: 前端路由刷新404？

**原因：** SPA应用需要服务器配置rewrite规则

**解决方案：**
1. OSS静态网站托管已配置error页为index.html
2. 确保所有路由都指向index.html

### Q4: CORS跨域问题？

**解决方案：**
1. 后端FC配置CORS中间件
2. OSS配置CORS规则
3. 确保OPTIONS预检请求通过
